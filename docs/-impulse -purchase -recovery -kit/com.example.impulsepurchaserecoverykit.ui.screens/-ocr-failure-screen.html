OcrFailureScreen

@Composable
fun OcrFailureScreen(paddingValues: PaddingValues, reason: OcrFailureReason, onTryAgain: () -> Unit, onManualEntry: () -> Unit, onDismiss: () -> Unit)

Error screen displayed when the receipt scanning or parsing pipeline fails.

Shows a contextual error message, emoji, and explanation tailored to the specific OcrFailureReason — so the user understands exactly what went wrong and what they can do about it. A tips card with photography advice is shown for all failure types to help the user avoid the same issue on their next scan.

Three action buttons are offered depending on the failure reason:

  • Try another image — always shown, navigates back to ScanScreen

  • Enter details manually — shown for OcrFailureReason.NO_TEXT_FOUND and OcrFailureReason.NO_ITEMS_PARSED only, as OcrFailureReason.INCOMPLETE_RECEIPT has already saved a partial receipt that the user can edit directly

  • Go back to home — always shown, dismisses the error and returns to HomeScreen

Parameters

paddingValues

Padding applied by the parent Scaffold to avoid overlap with system bars and the bottom navigation bar

reason

The OcrFailureReason that caused the scan to fail, used to select the appropriate message and button set

onTryAgain

Callback invoked when the user taps "Try another image" — navigates back to ScanScreen to select a new photo

onManualEntry

Callback invoked when the user taps "Enter details manually" — navigates to ManualEntryScreen to log the purchase by hand

onDismiss

Callback invoked when the user taps "Go back to home" — dismisses the error screen and returns to HomeScreen
