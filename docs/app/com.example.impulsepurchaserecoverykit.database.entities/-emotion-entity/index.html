EmotionEntity

data class EmotionEntity(val id: Long = 0, val receiptId: Long, val checkInTime: Long = System.currentTimeMillis(), val regretScore: Int, val mood: String, val notes: String?)

Room database entity representing a single emotion check-in for a receipt.

Every time a user completes the Rate Regret screen, a new EmotionEntity is created and stored in the emotion_checkins table. It captures the emotional state of the user at the time of rating — including their regret score, their selected mood, and any personal reflection note they chose to add.

Each EmotionEntity is linked to a parent ReceiptEntity via a foreign key. If the parent receipt is deleted, all associated emotion check-ins are automatically removed from the database via CASCADE deletion, keeping the database free of orphaned records.

An index is maintained on receiptId to ensure fast lookup of all emotion entries belonging to a specific receipt.
